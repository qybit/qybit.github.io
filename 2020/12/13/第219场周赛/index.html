T1：比赛中的配对次数

方法一：模拟

public int numberOfMatches(int n) {
        int res = 0;
        while (n > 1) {
            if (n % 2 == 0) {
                res += n/2;
                n /= 2;
            } else {
                res += (n-1)/2;
                n = (n-1)/2 + 1;
            }
        }
        return res;
 }

复杂度分析：

时间复杂度：$O(logN)$

空间复杂度：$O(1)$

T2：十-二进制数的最少数目

开头

对于 n = “123”

可以发现，最少也需要 k = 3 个，二进制串。123 = 111 + 11 + 1

由于 n 是十进制数，所以 k 最大值为 9。即 $0 \le k \le 9$

我们还可以发现，这几个二进制串之间是独立的，不存在进位的情况，所以对于任意一个十进制数，我们只需要找到其中的最大值即可。

代码

class Solution {
    public int minPartitions(String n) {
        int res = 0;
        for (char c : n.toCharArray()) res = Math.max(res, c-'0');
        return res;
    }
}

复杂度分析：

时间复杂度：$O(n)$

空间复杂度：$O(1)$

T3：石子游戏 VII

开头

首先，仍然是 Alice 先手。

这次的游戏规则是

  • 对这排石子，两人只能 移除 最左边或者最右边的石头。
  • 得分是计算 剩余所有石子的值。
  • Alice 的目标是 最大的扩大得分， Bob 的目标是 尽可能的减少得分的差距

如何理解尽可能的减少得分的差距，例如 stones=[4,3,5]

在 Alice 操作的情况下

  • 第一步移除 4，得分 5+ 3 = 8，此时 stones=[5,3]
  • Bob 操作时，Bob 会移除 3 ，所以 Bob 得分 5，此时 stones=[5]
  • 第二步移除 5，得分 0

所以在 Alice 的操作下，最大得分为 8，Bob 得分 5。此时的得分差距是 3

方法一：动态规划

这是属于 博弈DP 的一种，博弈中决策的出发点：就是在最坏的情况中选择最好的结果

定义 $f[i][j]$ 为区间 $[i, j]$ 先手减去后手的最大差值。

定义 $s[i]$ 为区间 $[0, i]$之间的区间和。

下面开始推状态转移方程：

从 Alice 视角开始出发，当 Alice 面对的局面时 $[i, j]$ 时

  • 当 Alice 移除最左边的元素时，此时 Alice 的得分 $Sum[i, j]$，而下一手对于 Bob 而言，他面对的情况就是 $[i+1, j]$
  • 当 Alice 移除最右边的元素时，此时 Alice 的得分是 $Sum[i-1, j-1]$，而下一手对于 Bob 而言，他面对的情况就是 $[i, j-1]$
  • Alice 面对的最坏的情况就是留给 Bob 的最好的情况，我们需要选取两者中最高的得分。

代码

class Solution {
    public int stoneGameVII(int[] stones) {
        int n =  stones.length;
        int[] s = new int[n + 1];
        for (int i = 1; i <= n; i++) s[i] = s[i-1] + stones[i-1];
        
        int[][] f = new int[n+1][n+1];
        // 区间
        for (int len = 2; len <= n; len++) {
            // 左右边界
            for (int i = 1; i + len - 1 <= n; i++) {
                int j = i + len - 1;
                f[i][j] = Math.max(s[j]-s[i]-f[i+1][j], s[j-1]-s[i-1]-f[i][j-1]);
            }
        }
        return f[1][n];
    }
}

复杂度分析：

时间复杂度：$O(n^2)$

空间复杂度：$O(n^2)$

T4：堆叠长方体的最大高度

开头

任意一个长方体是可以随意沿着 x, y, z 进行旋转操作的，我们必须要实现，从最底层开始长方体的高是逐渐递减的。

方法一：动态规划

先把每个长方体的长宽高升序排序，然后再以第一个元素，第二个元素，第三个元素优先级升序排序。

代码

cpp

class Solution {
public:
    int maxHeight(vector<vector<int>>& w) {
        for (auto &x : w) sort(x.begin(), x.end());
        sort(w.begin(), w.end(), greater<vector<int>>());
        int n = w.size();
        vector<int> f(n);
        int res = 0;
        for (int i = 0; i < n; i++) {
            f[i] = w[i][2];
            for (int j = 0; j < i; j++) {
                if (w[j][0] >= w[i][0] && w[j][1] >= w[i][1] && w[j][2] >= w[i][2]) {
                    f[i] = max(f[i], f[j]+w[i][2]);
                }
            }
            res = max(res, f[i]);
        }
        return res;
    }
};

java

class Solution {
    public int maxHeight(int[][] cuboids) {
        for (int[] x : cuboids) {
            Arrays.sort(x);
        }
        Arrays.sort(cuboids, ((o1, o2) -> {
            if (o1[0] == o2[0]) {
                if (o1[1] == o2[1]) {
                    return o2[2] - o1[2];
                }
                return o2[1] - o1[1];
            }
            return o2[0] - o1[0];
        }));
        //  for (int[] ints : cuboids) {
        //     System.out.println(Arrays.toString(ints));
        // }
        int n = cuboids.length;
        int[] f = new int[n];
        int res = 0 ;
        for (int i = 0; i < n; i++) {
            f[i] = cuboids[i][2];
            for (int j = 0; j < i; j++) {
                if (cuboids[j][0] >= cuboids[i][0] && cuboids[j][1] >= cuboids[i][1] 
                && cuboids[j][2] >= cuboids[i][2]) {
                    f[i] = Math.max(f[i], f[j] + cuboids[i][2]);
                }
            }
            res = Math.max(f[i], res);
        }
        return res;
    }
}

复杂度分析

时间复杂度：$O(N^2logN)$

空间复杂度：$O(N)$
