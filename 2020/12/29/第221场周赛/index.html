第221场周赛

  2020-12-29   力扣 算法 周赛   字符串 堆 贪心 动态规划 位运算 字典树 0  评论 字数统计: 2236字 阅读时长: 5分

题单：

  • 判断字符串的两半是否相似
  • 吃苹果的最大数目
  • 球会落何处
  • 与数组中元素的最大异或值

T1：判断字符串的两半是否相似

方法一：模拟

同时遍历，字符串的前一部分和后一部分，统计出现的元音字符，然后判断相等即可

代码

c++

class Solution {
public:
    bool halvesAreAlike(string s) {
        unordered_set<char> ss({'a', 'e', 'i', 'o', 'u'});
        int s1 = 0, s2 = 0, n = s.size();
        for (int i = 0; i < n/2; i++) {
            char c1 = tolower(s[i]);
            char c2 = tolower(s[n-1-i]);
            if (ss.count(c1)) s1++;
            if (ss.count(c2)) s2++;
        }
        return s1 == s2;
    }
};

复杂度分析

时间复杂度：$O(n)$

空间复杂度：$O(1)$，unordered_set 空间是固定大小

T2：吃苹果的最大数目

方法一：贪心 + 优先队列

开头

观察数据范围，可以发现是 $10^4$ 级别的。$1 \le n \le 2 * 10^4$ ，$0 \le days[i] \le 2 * 10^4$，所以我们最多可以吃掉 $4 * 10^4$ 个苹果。算法时间复杂度得保证为 $O(NlogN) $ 或者 $O(N)$。

思路

贪心策略：对于任意一天 i ，我们都应该选择 吃最先要过期的苹果 。

那么如何证明贪心解是符合最优解呢？将贪心解替换为最优解

假设在某一天 i 时，

对于贪心解，我们选择吃掉的是 $T_i $ 天要过期的苹果，而最优解吃掉的是 $T_j$ 天要过期的苹果。对于贪心解我们的策略是吃掉要过期的苹果，因此可以假设 $T_i \le T_j$ ，把贪心解替换为 $T_j$，由于此时的 $T_i \le T_j$ 的，所以贪心是符合的。所以 $T_j$ 替换 $T_j$ 是可行的，因此贪心是符合最优解的。

在构思代码时，需要借助优先队列这一数据结构实现每次吃最先要过期的苹果，以此降低时间复杂度。

代码

c++

class Solution {
public:
    int eatenApples(vector<int>& apples, vector<int>& days) {
        typedef pair<int, int> pii;
        // 小顶堆
        priority_queue<pii, vector<pii>, greater<pii>> heap;
        int n = apples.size();
        int res = 0;
        // 本题目，最多可以吃 40000 个苹果
        for (int i = 0; i <= 40000; i++) {
            // 今天有长出的新苹果，加入优先队列
            if (i < n && apples[i] > 0) 
                heap.push({i + days[i] - 1, apples[i]});
            
            // 把过期的苹果都扔掉
            while (heap.size() && heap.top().first < i) heap.pop();
            if (heap.empty()) continue;
            // 今天有苹果可以吃
            auto top = heap.top();
            heap.pop();
            res++;
            // 如果未过期就加入队列
            if (--top.second)
                heap.push(top);
        }
        return res;
    }
};

复杂度分析

时间复杂度：$O(NlogN)$，优先队列调整子节点，时间复杂度为 $O(logN)$，一共有 $N = 40000$次。

空间复杂度：$O(N)$

T3：球会落何处

方法一：模拟

对于网格的任意一栏 $column_i$，我们在第一行的所有列都放下一个球，然后记录从该 $column_i$栏放下球后，球的最终出口位置 $column_j$。如果不能到达最后一行，则返回 -1。

那么分析之后，球在下落的过程中，有以下几种情况：

  1. 这种情况下，球会被卡死，我们记录为 -1

  2. 在边界的情况下，球也会被卡死

在以上三种情况下，球才会卡死。我们在模拟下落时，避开这三种情况即可。

代码

class Solution {
public:
    vector<int> findBall(vector<vector<int>>& grid) {
        int m = grid.size(), n = grid[0].size();
        vector<int> res(n);
        for (int i = 0; i < n; i++) {
            int x = 0, y = i;
            while (x < m && y < n) {
                if (grid[x][y] == 1) {
                    if (y == n -1 || grid[x][y+1] == -1) break;
                    else x++, y++;
                } else {
                    if (y == 0 || grid[x][y-1] == 1) break;
                    else x++, y--;
                }
            }
            if (x >= m) res[i] = y;
            else res[i] = -1;
        }
        return res;
    }
};

复杂度分析

时间复杂度：$O(N^2)$

空间复杂度：$O(N)$

T4：与数组中元素的最大异或值

开头

观察数据的规模，确认是 $10^5$ 级别的数据量。因此任何 $O(N^2)$ 的算法，都不可能通过本题。但是，这里仍然给出 $TLE$ 代码，然后再后续优化算法。

方法一：暴力（TLE）

直接暴力循环，枚举所有的情况。

class Solution {
public:
    vector<int> maximizeXor(vector<int>& nums, vector<vector<int>>& queries) {
        int n = nums.size(), m = queries.size();
        vector<int> ans(m);
        for (int i = 0 ;i < m; i++) {
            auto query = queries[i];
            int x = query[0], m = query[1];
            int res = INT_MIN, f = 0;
            for (int j = 0; j < n; j++) {
                if (nums[j] <= m) {
                    f = 1;
                    res = max(res, nums[j] ^ x);
                }
            }
            ans[i] = f ? res : -1;
        }
        return ans;
    }
};

复杂度分析

时间复杂度：$O(N^2)$

空间复杂度：$O(N)$

方法二：贪心 + 前缀树（Trie）

开头

异或规则：相同为0，不同为1。

本题 $0 \le nums[i] \le 10^9$，所以二进制数一共有 31 位。本题的前缀树和经典的前缀树不同，该题中要构造的前缀树的子节点仅含 0 和 1，是一个数从最高位开始，向下延申。因此对于 $10^5$ 且 数据本身为 $10^9$ 的数据量，我们需要构造的字典树大小为 3100000 * 2 这是最为保险的估计量。

在本题，我们需要计算出在不大于 $queries[i][1]$ 的情况下，计算每一个查询中$max \{ queries[i][0] \ \ XOR \ \ nums[j] \}$。那么根据异或规则，我们要尽可能的让每一位都不同，这样才能让结果产生更多的1，那么结果就更大。

比如$A = 101010101 …. 101$ 时，假设 $A$ 为某一个 31 位二进制数，我要尽可能在不大于 $queries[i][1]$ 的集合中，找出这样一个数 $$ A = 101010101 …. 101 \\
B = 010101010 …. 010 \\
res = 111111111 …. 111 $$ 在理想情况下，我们可以找到一个 $B$ 满足恰好每一位都与 $A$ 都是相反的，得到的 $res$ 所有位全为 1。

现实情况是，我们要尽可能找到一个 $B$ ，使得 $B$ 中有更多位满足上述情况。这是贪心策略的体现。

对于前缀树，对于任何一位上的 0 或者 1，我们的选择树如下：

root 表示为 查询数 $X$ 的某一位，如果 $root == 1$ ，在子树 0 存在的下，我们将会抛弃子树 1 来剪枝。那么对于原本的集合，我们将会去掉 1 那部分。

对于集合的另一部分，0 也是同理，不断缩小集合范围，最终找到最大的 $XOR$ 值。

值得注意的是，本题中是已经给出了所有的查询，因此查询本身的顺序并不重要，所以我们可以对查询进行预处理。

代码

c++

struct Node {
    int x, m, k;
    bool operator < (Node& t) const {
        return m < t.m;
    }
}q[100010];

int son[3100000][2];

class Solution {
public:
    int idx = 0;
	// 从最高位 开始插入
    void insert(int x) {
        int p = 0;
        // 数据本身 最高31 位，因此移动30位
        for (int i = 30; ~i; i--) {
            int u = x >> i & 1;
            // 不存在路径，就创建
            if (!son[p][u]) son[p][u] = ++idx;
            p = son[p][u];
        }
    }

    int search(int x) {
        // 路径不存在
        if (!idx) return -1;
        int res = 0, p = 0;
        for (int i = 30; ~i; i--) {
            // 取反去最高位
            int u = !(x >> i & 1);
            // 该位 贡献一个1
            if (son[p][u]) p = son[p][u], res += 1 << i;
            // 相反的数，不存在，只能去相同的路径
            else p = son[p][!u];
        }
        return res;
    }

    vector<int> maximizeXor(vector<int>& nums, vector<vector<int>>& queries) {
        idx = 0;
        memset(son, 0, sizeof son);
        int n = nums.size(), m = queries.size();
        for (int i = 0; i < m; i++) {
            q[i] = {queries[i][0], queries[i][1], i};
        }
        // 预处理
        sort(q, q+m);
        sort(nums.begin(), nums.end());
        vector<int> res(m);
        for (int i = 0, j = 0; j < m; j++) {
            while (i < n && nums[i] <= q[j].m) insert(nums[i++]);
            res[q[j].k] = search(q[j].x);
        }
        return res;
    }
};

复杂度分析

时间复杂度：$O(NlogN) + O(MlogM)$，其中 N 为数组长度， M 为查询次数。

空间复杂度：$O(312N + M)$，其中31表示 int 范围内，二进制数的位数。M 为查询次数
