示例代码：c++

T1：重新格式化电话号码

方法一：模拟

我们先去掉所有的“-” 和 “ ” 。尽可能的每次取三个数，然后直到仅剩下4个数时，停止取数，然后根据分块规则进行格式化即可。

代码

class Solution {
public:
    string reformatNumber(string number) {
        string s;
        for (auto c : number)
            if (c != ' ' && c != '-')
                s += c;
        
        string res;

        for (int i = 0 ; i < s.size(); ) {
            if (s.size() - i > 4) {
                res += s.substr(i, 3);
                i += 3;
            } else {
                int k = s.size() - i;
                if (k <= 3) res += s.substr(i, k);
                else res += s.substr(i, 2) + "-" + s.substr(i+2, 2);
                i += 4;
            }
            res += "-";
        }
        res.pop_back();
        return res;
    }
};

复杂度分析

时间复杂度：$O(n)$

空间复杂度：$O(n)$

T2：删除子数组的最大得分

开头

我们需要删除一个含有 若干个不同元素 的子数组，也就是说我们要找出数组和最大的无重复元素的连续子数。而且看数据范围提示， $nums[i]$ 全部都是正数，因此这个不重复子数组只要足够长，那么它就是最大得分。

方法一：双指针

移动规则：

  • 右指针是不断向右边扩展的，此时需要记录当前区间的和
  • 当遇到重复的元素时，左指针再收缩
  • 关于去重，我们可以使用哈希表来记录

代码

class Solution {
public:
    int maximumUniqueSubarray(vector<int>& nums) {
        int n = nums.size();
        // 无序 hash表，效果类似于 Java中的HashMap
        unordered_map<int, int> hash;
        int res=  0;
        for (int i = 0, j = 0, s=  0; i < n ; i++) {
            int x=  nums[i];
            hash[x]++;
            s += x;
            while (hash[x] > 1) {
                s -= nums[j];
                hash[nums[j++]]--;
            }
            res = max(res, s);
        }
        return res;
    }
};

复杂度分析

时间复杂度：$O(n)$

空间复杂度：$O(n)$

方法二：前缀和+哈希表

思路：

提前计算好前缀和，再借助哈希表来记录上一次元素的出现位置，依次计算当前位置与上一次出现位置的区间和。

算法表述图如下

假设某一个元素的当前位置为 $i$，那么在区间 $[0, i]$ 之间存在一个位置 $j$，$j$ 对应的元素等于当前位置 ，此时计算出区间和 $Sum_{i+1} - Sum_{j+1}$ ，其中 $Sum$ 为前缀和。

代码

class Solution {
public:
    int maximumUniqueSubarray(vector<int>& nums) {
        int n = nums.size();
        vector<int> s(n+1);
        for (int i = 1; i <= n; i++) s[i] = s[i-1] + nums[i-1];

        int j = 0, res = 0;
        unordered_map<int, int> f;
        for (int i = 0; i < n; i++) {
            int x = nums[i];
             if (f.find(nums[i]) != f.end() && j <= f[nums[i]]) {
                j = f[nums[i]] + 1;
            }

            res = max(res, s[i+1] - s[j]);
            f[x] = i;
        }
        
        return res;
    }
};

复杂度分析

时间复杂度：$O(n)$

空间复杂度：$O(n)$

T3：跳跃游戏 VI

方法一：动态规划（可能TLE）

我们设置 $f[i]$ 为从 0 到 $i$ 能拿到的最大得分。

那么假设，此时我们在任意一点 $i$ 处时，此时方案一共有 $i - j = k$ 个

那么不难推出动态转移方程为： $$ f[i] = max{f[k]} + nums[i]，其中 i \le k \le j \ \
f[0] = nums[0]，第一个元素是必选的 $$ 那么根据数据规模的 $10^5$ 来看，我们的这个方程式对应的算法时间复杂度为 $O(n*k)$ 。所以此方法是可能会 TLE 的。

方法二：动态规划+单调队列 优化

根据上述的公式，可以发现在状态转移的过程中，最耗时的部分就是在区间 $[j, i]$，找出最大值。

这个动作，我们可以借助单调队列来实现 $O(1)$ ，以此来进行时间上的优化，相对于原本的 $O(k)$ 而言是绝对可以通过的。

单调队列

我们可以借助 双端队列(Deque) 实现，假设区间长度为 $k$ 时：

  • 当遍历当前元素位置大于区间长度 $k$ 时，我们清理掉所有小于 $i - k$ 位置的元素
  • 然后再计算对头最大元素即对应方程中 $max{f[k]}, i \le k \le j$
  • 删除所有小于等于 $f[i]$ 的元素，保证 $deque$ 的递减性
  • 再把 $i$ 加入到 $deque$

代码

class Solution {
public:
    int maxResult(vector<int>& nums, int k) {
        int n = nums.size();
        vector<int> f(n);
        f[0] = nums[0];
        deque<int> q;
        q.push_back(0);
        for (int i = 1; i < n; i++) {
            while (q.front()< i - k) q.pop_front();
            f[i] = f[q.front()] + nums[i];
            while (q.size() && f[q.back()] <= f[i]) q.pop_back();
            q.push_back(i);
        }
        return f[n-1];
    }
};

复杂度分析

时间复杂度：$O(n)$

空间复杂度：$O(n)$

T4：检查边长度限制的路径是否存在

方法一：排序 + 并查集

开头

题目要求，给出一个无向图的边集，然后提前给出多组查询条件（可以理解为离线）。也就是说，可以不必按照特定的顺序进行查询。我们进行排序操作后，对时间复杂度有一定的把握。

与 离线 对应的 在线，即一次只给出一组查询，必须按照给出顺序进行查询。这样的操作会使时间复杂度无法确定。

思路：

本题是给出了所有的查询要求，所以基于 “缓存” 的思想，我们可以按照 $queries$ 的 $limit_j$ 来进行排序。

同样的，我们可以对边长 $edgeList$ 的 $dis_i$ 进行排序。

我们处理所有的询问时，借助并查集维护图的连通性，对于 $queries_i = (p_i, q_i, limit_i)$ 时，我们需要不断的向并查集中添加 $edgeList_j$ $dis_j \le limit_i$ 的边，然后再验证 $p_i 和 q_i$ 连通性。

const int N = 100010;

struct Node {
    // a -> 起点；b -> 终点；c -> dis；d -> 原 queries_i 位置
    int a, b, c, d;
    // 重载 < 为了方便排序操作，如果不重载，请传入 lambda表达式 或者 cmp 函数
    bool operator < (const Node & t) const {
        return c < t.c;
    }
} e[N], q[N];


class Solution {
public:
    vector<int> f;

    int find (int x) {
        if (x != f[x]) f[x] = find(f[x]);
        return f[x];
    }

    bool is_connected(int x, int y) {
        return find(x) == find(y); 
    }

    vector<bool> distanceLimitedPathsExist(int n, vector<vector<int>>& ee, vector<vector<int>>& qq) {
        int m = ee.size(), k = qq.size();
       

        for (int i = 0; i < m; i++) 
            e[i] = {ee[i][0], ee[i][1], ee[i][2]};
        for (int i = 0; i < k; i++) 
            q[i] = {qq[i][0], qq[i][1], qq[i][2], i};
        // 排序
        sort(e, e + m), sort(q, q + k);
        // 初始化 并查集 
        f.resize(n);
        for (int i = 0; i < n; i++) f[i] = i;
        // 结果数组
        vector<bool> res(k);
        for (int i = 0, j = 0; i < k; i++) {
            // 不断的添加边 直到 dis_j >= limit_i
            while (j < m && e[j].c < q[i].c) {
                int a = e[j].a, b = e[j].b;
                f[find(a)] = find(b);
                j++;
            }
            // 验证连接性
            res[q[i].d] = is_connected(q[i].a, q[i].b);
        }
        return res;
        
    }
};

复杂度分析

时间复杂度：$O(NlogN)$

空间复杂度：$O(n)$
