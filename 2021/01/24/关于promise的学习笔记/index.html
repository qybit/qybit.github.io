关于Promise的学习笔记

  2021-01-24   前端   前端 Promise 0  评论 字数统计: 1265字 阅读时长: 3分

Promise 的出现是为了解决 “回调地狱”，这种重度嵌套回调函数的代码风格而出现的。如下图

借助 Promise ,我们就可以避免这种风格的代码，从而让代码变成丝滑如顺链式编程。如

function1()
    .then(function2())
    .then(function3())
	.catch(functionx())

一、项目结构

假设我们被迫有如下业务需求，先查询 user.json 文件，找到 某个用户的id，然后再查询 这个用户的对应的课程id，然后再根据课程id，查询得到课程对应的分数。

二、正常写法

能写出这样的代码，在经常写 JavaScript，恐怕并不会觉得很别扭，但是实际上这样的代码并不利于后期的维护和修改。

如果让写这种代码的人自己看的话，然后再修改代码逻辑(hhhh~)，估计本人也想抽自己两巴掌。

//1、查出当前用户信息
//2、按照当前用户的id查出他的课程
//3、按照当前课程id查出分数
$.ajax({
	url: "mock/user.json",
    // 成功找到用户
    success(data) {
        console.log("查询用户：", data);
        $.ajax({
            url: `mock/user_corse_${data.id}.json`,
            // 成功找到对应课程
            success(data) {
                console.log("查询到课程：", data);
                $.ajax({
                    url: `mock/corse_score_${data.id}.json`,
                    // 成功找到课程对应的分数
                    success(data) {
                        console.log("查询到分数：", data);
                    },
                    error(error) {
                        console.log("出现异常了：" + error);
                    }
                });
            },
            error(error) {
                console.log("出现异常了：" + error);
            }
        });
	},
	error(error) {
		console.log("出现异常了：" + error);
	}
});

三、Promise 写法

我们可以发现，已经具有链式编程的雏形了，但是代码依旧是十分的难读。而且现在的代码还存在大量重复的部分，我们可以把重复的部分抽出来，再做进一步的封装。

// 用 Promise 对象，封装异步操作
let p = new Promise((resolve, reject) => {
	//1、异步操作
     $.ajax({
        url: "mock/user.json",
        success: function (data) {
             console.log("查询用户成功:", data)
             resolve(data);
        },
         error: function (err) {
             reject(err);
        }
     });
});

p.then((obj) => {
     return new Promise((resolve, reject) => {
         $.ajax({
            url: `mock/user_corse_${obj.id}.json`,
             success: function (data) {
                 console.log("查询用户课程成功:", data)
                 resolve(data);
             },
             error: function (err) {
                 reject(err)
             }
         });
     })
 }).then((data) => {
     console.log("上一步的结果", data)
     $.ajax({
         url: `mock/corse_score_${data.id}.json`,
         success: function (data) {
             console.log("查询课程得分成功:", data)
         },
         error: function (err) {
         }
     });
})

四、封装 Promise

思考一个问题：如何才能让 Promise 对象不停息的调用下去呢？

我们借鉴很多地方，比如 Java 中 StringBuilder#append 方法的写法，它这里的逻辑是，每次 append 操作后，将自身返回。那么操作者可以选择继续 append，或者是停止调用。

@Override
public StringBuilder append(char c) {
    super.append(c);
    return this;
}

我们只需要封装一个入口函数，让这个函数每次返回一个Promise 对象。

const get = (url, data) => {
    return new Promise((resolve, reject) => {
        $.ajax({
            url: url,
            data: data,
            success: function (data) {
                resolve(data);
            },
            error: function (err) {
                reject(err)
            }
        })
    });
}

然后，我们的业务就可以变成这样了。把 then 理解为事件A，当事件A结束后，可以选择进一步做事件B，或者 尝试catch 错误而中止调用。 A -> B / error -> C / error

get("mock/user.json")
    .then((data) => {
        console.log("用户查询成功~~~:", data)
        return get(`mock/user_corse_${data.id}.json`);
    })
    .then((data) => {
        console.log("课程查询成功~~~:", data)
        return get(`mock/corse_score_${data.id}.json`);
    })
    .then((data)=>{
        console.log("课程成绩查询成功~~~:", data)
    })
    .catch((err)=>{
    	console.log("出现异常",err)
	});
