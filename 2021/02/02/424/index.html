424 替换后的最长重复字符

时间： 2021-02-02   |   分类： 力扣   | 字数： 420 字 | 阅读： 1分钟 | 阅读次数：

力扣424链接：替换后的最长重复字符

方法一：双指针

我们枚举每一个字符作为区间右端点，然后找到相同字符对应的最远左端点，统计左右端点区间内存在的不同于当前字符的数量。那么此时的区间，就是一个符合答案的区间。

我们继续尝试扩大区间，右端点移动一格。当不满足条件时（即区间内非当前字符的数量大于K），我们将左端点至多移动一格。移动过程中，右端点是不断右移，左端点只有不满足条件时移动。

代码思路

由于仅包含大写字母，所以我们可以使用一个长度26的数组来统计字符的出现次数

使用一个变量记录下，历史的最大区间长度

代码

class Solution {
public:
    int characterReplacement(string s, int k) {
        int left = 0, right = 0;
        vector<int> hash(26);
        int n = s.size();
        int mx = 0, res = 0;
        while (right < n) {
            char c1 = s[right];
            hash[c1 - 'A']++;
            mx = max(mx, hash[c1 - 'A']);
            if (right - left - mx + 1 > k) {
                char c2 = s[left];
                hash[c2 - 'A']--;
                left++;
            }
            right++;
            res = max(res, right - left);
        }
        return res;
    }
};

复杂度分析

时间复杂度：$O(N)$，其中 $N$ 为字符串 s 的长度。我们至多会遍历一次字符串

空间复杂度：$O(|\sum|)$ ，其中 $\sum$ 为各个字符出现次数的总和。我们要统计所有大写字符的数量
